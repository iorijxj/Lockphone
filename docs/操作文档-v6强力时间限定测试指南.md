# Lockphone v6 强力时间限定 真机测试指引（给现场操作人）

> 这份文档写给**不懂开发的操作人**。照着一步一步做，每做一步把结果记下来（截图或复制文字），填到对应的「输出结果」里，最后统一发回。
> 遇到和文档说的不一样的情况：**停下来，把屏幕拍照发回**，不要自己猜着继续。

**这次要做的事**：这台 vivo NEX S 之前已经装好并激活过「儿童锁定桌面（Lockphone）」。这次要
1. **更新**成带「强力时间限定」的新版本程序；
2. 给它开一个新权限（使用情况访问）；
3. 测试新功能：**受限 APP 用够设定时间后会被自动锁住、打不开，直到家长加时或第二天自动恢复**。

**⚠️ 三条铁律（先读三遍）**
1. **绝对不要点「恢复出厂设置」「清除所有数据」「格式化」**——手机里的学习进度不能丢
2. 文档里的命令**原样复制粘贴**执行，不要手打（容易打错）
3. 每条命令执行后**把黑窗口里的输出文字复制保存**，最后要发回

## 第 0 步：准备清单

| 需要的东西 | 说明 |
|---|---|
| 上次那台 Windows 电脑 | `C:\adb\platform-tools\adb.exe` 应该还在，直接用 |

## 第 1 步：连接手机，确认能通信

1. 数据线连电脑，手机若弹「USB 用途」选「**传输文件**」
2. 打开命令窗口：`Win + R` → 输入 `powershell` → 回车
3. 执行：

```
C:\adb\platform-tools\adb.exe devices
```

预期看到一串数字字母后面跟 `device`：

```
List of devices attached
4d57f77b        device
```

- 显示 `unauthorized` → 回手机点「一律允许」弹窗，再执行一遍
- 什么都不显示 → 换数据线 / 换 USB 口

## 第 2 步：更新安装新版本程序

> 这是「覆盖更新」，**不会清数据、不会掉管理权限**，上次设的 PIN 和白名单都还在。

```
C:\adb\platform-tools\adb.exe install -t -r C:\adb\app-debug.apk
```

⚠️ 命令里的 `-t` 和 `-r` 都不能省：`-t` 允许测试包，`-r` 表示覆盖更新保留数据。

预期输出：`Success`

常见失败及处理：
| 报错里包含 | 处理 |
|---|---|
| `INSTALL_FAILED_ABORTED: User rejected permissions` | 重新执行命令，执行后**立刻看手机屏幕**，弹出的「继续安装/允许」点掉（可能要输 vivo 账号密码） |
| `INSTALL_FAILED_TEST_ONLY` | 漏了 `-t`，用上面完整命令重来 |
| `INSTALL_FAILED_VERSION_DOWNGRADE` | 新包版本号比手机上的低，把情况发回等新包 |
| `signatures do not match` | 新包和旧包签名不一致，**停下发回**，不要卸载重装（会丢数据），等开发方处理 |
| 其他报错 | 复制完整报错发回，等回复再继续 |

**【记录】**输出复制保存。

## 第 3 步：授予「使用情况访问」权限（关键新步骤）

> 时间限定要靠这个权限来判断"孩子现在正在用哪个 APP"。**不开这个权限，计时完全不生效**，等于没做。

执行授权命令：

```
C:\adb\platform-tools\adb.exe shell appops set com.lockphone android:get_usage_stats allow
```

这条命令**执行成功时没有任何输出**（不报错就是成功）。

紧接着执行验证命令，确认真的开了：

```
C:\adb\platform-tools\adb.exe shell appops get com.lockphone android:get_usage_stats
```

预期输出里包含 `allow`，类似：

```
android:get_usage_stats: allow
```

- 如果显示 `deny` 或 `default` → 重新执行上面的授权命令；仍不行则发回

**【记录】**两条命令的输出都要发回。

## 第 4 步：确认管理权限还在，打开程序

1. 确认管理权限仍激活（这条会从系统状态里筛出关键几行）：

```
C:\adb\platform-tools\adb.exe shell dumpsys device_policy | findstr /C:"Device Owner" /C:"com.lockphone"
```

预期至少能看到一行含 `Device Owner`、以及一行含 `com.lockphone`，类似：

```
  Device Owner:
    admin=ComponentInfo{com.lockphone/com.lockphone.admin.LockAdminReceiver}
    package=com.lockphone
```

- 如果**什么都没输出**（空白），说明管理权限掉了，**停下发回**（正常更新不该掉）。
- 如果 `findstr` 那句报错或不认，改用不带筛选的完整命令 `C:\adb\platform-tools\adb.exe shell dumpsys device_policy`，把开头 `Device Owner` 附近的内容复制发回。

2. 手机上打开 **Lockphone**，应正常进入锁定桌面（顶部显示剩余电量，右上有「家长模式」）。

## 第 5 步：设置一个"短配额"方便快速测试

> 正式用时配额会设成几十分钟；测试时先设 **1 分钟**，好在一分钟内看到效果。

1. 手机上桌面点右上「**家长模式**」→ 输入家长 PIN
2. 进入家长设置，往下找到**白名单列表**。挑一个**无所谓被临时锁住的 APP**当测试对象（**别选正在用的学习 APP**，随便一个已勾选白名单的即可，比如浏览器）
3. 确认它前面是**已勾选**状态（打勾才会出现在孩子桌面、才有限时选项）
4. 在它下面：打开「**限时**」开关 → 在「分钟」框里把数字改成 **1**
5. 顺便记下这个测试 APP 的名字
6. 点「返回桌面」

> 顺便验证一下第 3 步的权限：进家长设置时**顶部不应出现**红字「⚠ 未授予使用情况访问权限」。如果出现了红字，说明第 3 步没成功，回第 3 步重做。

**【记录】**记下测试用 APP 的名字；家长设置页顶部有没有红字提示。

## 第 6 步：核心测试——时间到自动锁住

> 这是本次最重要的验证：**堵住"APP 一直开着不退出就能无限用"的老漏洞**。

1. 在孩子桌面点开刚才设了 1 分钟的测试 APP
2. **就一直用它、别退出**（正常玩、翻页、点点点），盯着计时
3. 大约 **1 分钟**（可能多几秒）后观察会发生什么：
   - 预期：APP 被自动踢出 / 或弹出系统「应用已暂停」之类的提示，并弹一条 Toast「**「XX」今天的时间用完啦**」
   - 回到 Lockphone 桌面后，这个 APP 的图标应该**变灰**、名字变成「**今日已用完**」，**点它没有反应，打不开**

4. **如实记录实际现象**（这一项很关键，务必描述清楚）：
   - 到时间后，APP 是被**直接踢回**了 Lockphone 桌面，还是**停在原界面弹了个系统提示**？还是**没反应、还能继续用**？
   - 有没有看到那条 Toast 文字？
   - 回到桌面后，那个 APP 图标是不是变灰、显示「今日已用完」、点不动？

| # | 测试动作 | 预期结果 | 实际(✓/✗) |
|---|---|---|---|
| 1 | 用满 1 分钟不退出 | 到点被锁，弹「今天的时间用完啦」 | |
| 2 | 回桌面看该 APP 图标 | 变灰 + 显示「今日已用完」 | |
| 3 | 点那个变灰的图标 | 打不开、没反应 | |

**【记录】**上面表格 + 第 4 点的现象详细描述（文字或录屏）。

## 第 7 步：测试家长加时（把锁住的 APP 放开）

1. 桌面点右上「**家长模式**」→ 输入 PIN
2. 在白名单列表找到刚被锁住的那个测试 APP，它下面会显示「今日已用 1 / 1 分（已用完）」
3. 两种加时都试一下：
   - 点「**+5分**」按钮
   - 或在「加时」框里输入一个数字（比如 `3`）再点「加时」
4. 点「返回桌面」，看那个 APP 图标：预期**恢复正常**（不灰了、名字变回 APP 名），**能重新打开**
5. 打开它继续用，验证加的时间用完后会**再次被锁**

## 第 11 步（可选，谨慎）：测试第二天自动恢复

> ⚠️ 这步要**临时改手机日期**。改日期可能影响某些看重"连续打卡"的 APP（如多邻国的连胜天数），介意就**跳过这步**（第二天自然会自动恢复，不测也行）。做的话测完**务必把日期改回自动**。

1. 先让测试 APP 处于「今日已用完」被锁状态
2. 手机设置 → 系统管理 → 日期和时间 → **关闭「自动确定日期和时间」** → 手动把日期改到**明天**
3. 回 Lockphone 桌面等约 5–10 秒，看被锁的 APP：预期**自动恢复正常、可打开**，且家长设置里「今日已用」清零
4. **测完立刻**：设置 → 日期和时间 → **重新打开「自动确定日期和时间」**

## 出问题的万能救援命令

手机被锁死退不出来、或程序异常时，连上电脑执行（**数据不受影响**）：

解除管理权限：
```
C:\adb\platform-tools\adb.exe shell dpm remove-active-admin com.lockphone/.admin.LockAdminReceiver
```

如需彻底卸载程序：
```
C:\adb\platform-tools\adb.exe uninstall com.lockphone
```

如需单独关掉本次新加的权限（一般不用）：
```
C:\adb\platform-tools\adb.exe shell appops set com.lockphone android:get_usage_stats deny
```
