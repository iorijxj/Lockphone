# Lockphone 真机预检操作指南（给现场操作人）

> 这份文档写给**不懂开发的操作人**。照着一步一步做，每做一步把结果记下来（截图或复制文字），最后统一发回。
> 遇到和文档说的不一样的情况：**停下来，把屏幕拍照发回**，不要自己猜着继续。

**这次要做的事**：在一台 vivo NEX S 手机上安装一个「儿童锁定桌面」的验证程序，并激活它的管理权限，验证锁定功能有效。

**⚠️ 三条铁律（先读三遍）**
1. **绝对不要点「恢复出厂设置」「清除所有数据」「格式化」**——手机里的学习进度不能丢
2. 文档里的命令**原样复制粘贴**执行，不要手打（容易打错）
3. 每条命令执行后**把黑窗口里的输出文字复制保存**，最后要发回

---

## 第 0 步：准备清单

| 需要的东西 | 说明 |
|---|---|
| 一台 Windows 电脑 | 操作用 |
| vivo NEX S 手机 | 就是要装锁定桌面的这台 |
| 一根**能传数据**的 USB 数据线 | 有些线只能充电不能传数据，连上没反应就换一根 |
| vivo 账号和密码 | 手机上登录的那个 vivo 账号，后面开开关、退登录都要用 |
| 手机锁屏密码 | 中途会用到 |
| 文件 `app-debug.apk` | 由发起人（开发方）提供，收到后放到 `C:\adb\` 文件夹（第 1 步会建这个文件夹） |

---

## 第 1 步：电脑上准备 adb 工具

adb 是谷歌官方的手机调试工具，绿色免安装。

1. 在电脑浏览器打开这个网址下载（约 5MB）：
   `https://dl.google.com/android/repository/platform-tools-latest-windows.zip`
2. 在 C 盘根目录新建文件夹 `C:\adb`
3. 把下载的 zip 解压到 `C:\adb`，解压后应该存在这个文件：
   `C:\adb\platform-tools\adb.exe`
4. 把收到的 `app-debug.apk` 也放进 `C:\adb`，即：`C:\adb\app-debug.apk`
5. 打开命令窗口：按键盘 `Win + R`，输入 `powershell`，回车，会弹出一个蓝底或黑底窗口。**后面所有命令都在这个窗口里执行。**
6. 粘贴执行下面这条命令验证工具可用（在窗口里点右键即粘贴，回车执行）：

```
C:\adb\platform-tools\adb.exe version
```

看到类似 `Android Debug Bridge version 1.0.41` 的输出就是成功。
**【记录】**把输出复制保存。

## 第 2 步：手机上打开开发者选项和 USB 调试

都在手机上操作（vivo 系统菜单名称可能略有差异，找不到就在设置顶部搜索框里搜关键词）：

1. **打开开发者选项**：设置 → 系统管理 → 关于手机 → 版本信息 → 找到「软件版本号」，**连续快速点击 7 次**，直到提示"您已处于开发者模式"
2. **打开 USB 调试**：设置 → 系统管理 → 开发者选项 → 打开「**USB 调试**」开关
3. **打开 USB 安装**：同一页面找到「**USB 安装**」开关并打开
   - ⚠️ vivo 会要求**登录 vivo 账号**才允许打开这个开关，按提示登录（这就是准备清单里要账号密码的原因）
   - 个别版本还要求手机插着 SIM 卡，如果提示了就插一张
4. 如果开发者选项里还有「**USB 调试（安全设置）**」开关，也一并打开

## 第 3 步：连接手机

1. 用数据线把手机连到电脑
2. 手机上如果弹出「USB 用途」选择，选「**传输文件**」
3. 电脑命令窗口执行：

```
C:\adb\platform-tools\adb.exe devices
```

4. **第一次执行时手机会弹窗**「是否允许 USB 调试？」→ 勾选「一律允许」→ 点允许。然后**再执行一遍**上面的命令。
5. 成功的样子（一串数字字母 + `device`）：

```
List of devices attached
1234567890ABCDEF        device
```

- 显示 `unauthorized` → 回手机上点允许弹窗
- 什么都不显示 → 换一根数据线 / 换一个 USB 口

**【记录】**把输出复制保存。


## 第 6 步：安装验证程序

```
C:\adb\platform-tools\adb.exe install -t -r C:\adb\app-debug.apk
```

⚠️ 命令里的 `-t` 不能省，省了必定失败。

预期输出：`Success`

常见失败及处理：
| 报错里包含 | 处理 |
|---|---|
| `INSTALL_FAILED_ABORTED: User rejected permissions` | 手机屏幕上弹出的「是否允许安装」确认窗没被点允许。**重新执行命令，执行后立刻看手机屏幕**，弹窗出来点「继续安装/允许」（可能要求输入 vivo 账号密码） |
| `INSTALL_FAILED_TEST_ONLY` | 命令里漏了 `-t`，用上面的完整命令重来 |
| `INSTALL_FAILED_USER_RESTRICTED` 或手机弹窗拒绝 | 回到第 2 步，确认「USB 安装」开关真的开了 |
| 其他报错 | 复制完整报错发回，等回复再继续 |

装好后手机桌面会出现一个叫 **Lockphone** 的应用图标。**先不要打开它。**

**【记录】**输出复制保存。

## 第 7 步：退出手机上所有账号

⚠️ 这一步是激活的前提：Android 系统硬性规定，激活管理权限时手机上**系统账号数量必须为 0**，一个都不能留，没有绕过办法。
⚠️ 只是「退出登录/移除挂靠条目」，**不是**删除数据、不是恢复出厂。

根据第 4 步的检查结果，这台手机上共有 **2 个系统账号**，都要处理：

1. **退出 vivo 账号**（`18650716330`）：设置 → 顶部点自己的账号头像 → 退出登录
   - 如果提示「请先关闭查找手机」：设置 → 安全（或搜索"查找手机"）→ 关闭「查找手机」，需要输入 vivo 账号密码，关完再退
2. **移除多邻国的系统挂靠账号**（`多邻国`）：设置 → 更多设置 → 账户与同步（或叫「账号」），找到「多邻国」→ 移除账户
   - 放心：移除的只是系统里的挂靠条目，**多邻国 APP 内的登录和数据不动**；而且多邻国的学习进度保存在它的服务器上，万一掉登录，重新登录同一个账号进度原样回来
3. **⚠️ 移除多邻国账号之后、第 8 步激活成功之前，不要打开多邻国 APP**——一打开它可能自动把账号重新注册回系统，导致激活失败。忍住，做完第 8 步再开。
4. 电脑上执行验证：

```
C:\adb\platform-tools\adb.exe shell dumpsys account
```

输出里第一行附近应显示 `Accounts: 0`，且没有任何 `Account {` 条目。如果还有，说明有账号没退干净，输出会写着它的名字，照着去设置里退掉。

5. 验证通过后**马上接着做第 8 步**（不要隔很久，防止某个应用悄悄把账号加回来）。

**【记录】**验证命令输出。

---

## 第 8 步：激活管理权限（关键一步）

```
C:\adb\platform-tools\adb.exe shell dpm set-device-owner com.lockphone/.admin.LockAdminReceiver
```

**成功的样子**：
```
Success: Device owner set to package ComponentInfo{com.lockphone/com.lockphone.admin.LockAdminReceiver}
```

**失败对照表**：
| 报错里包含 | 含义 | 处理 |
|---|---|---|
| `already some accounts on the device` | 还有账号没退干净 | 回第 7 步，用验证命令找出漏网账号 |
| `already several users` | 手机上有多用户/应用分身 | 设置里搜「多用户」「用户」「应用分身」，删掉除机主外的所有用户和分身后重试 |
| `Unknown admin` | 程序没装好 | 回第 6 步重装 |
| 其他报错 | — | 复制完整报错发回，**停下等回复** |

激活成功后，**现在可以打开多邻国了**：确认登录态和学习进度都正常（第 5 步的复查点）。它把系统账号加回去也没关系，激活已经完成，不受影响。

**【记录】**无论成败，输出复制保存发回；多邻国复查结果也记一下。**这一步失败且对照表解决不了，就到此为止等下一步指示，后面的不用做了。**

---

## 第 9 步：验证锁定效果

1. 手机上打开 **Lockphone**，屏幕上应显示「**Device Owner: 已激活**」
2. 点「**进入锁定**」按钮
3. 逐项测试并记录（✓ 有效 / ✗ 无效）：

| # | 测试动作 | 预期结果 | 实际(✓/✗) |
|---|---|---|---|
| 1 | 从屏幕顶部往下拉 | 通知栏拉不出来 | |
| 2 | 按最近任务键（或上滑悬停） | 没反应 |  |
| 3 | 按 Home 键（或上滑回桌面） | 还留在 Lockphone 里 | |
| 4 | 疯狂点返回键 | 退不出去 | |
| 5 | 长按电源键 | 能看到关机/重启菜单 | |
| 6 | 按电源键锁屏，再解锁 | 解锁后还在 Lockphone 里 | |

4. **重启测试**：长按电源键 → 重启。开机解锁后观察：是直接进入了 Lockphone，还是弹了「选择桌面」的窗口，还是回到了原来的桌面？**如实记录看到的现象。**
5. **安全模式测试**：长按电源键 → 在弹出的菜单上**长按「重启」**（vivo 进安全模式的方式，如果没反应就记录"无法触发"）→ 如果提示进入安全模式，看它是被拒绝还是真的进去了。**记录现象后正常重启回来。**

**【记录】**上面表格 + 第 4、5 项的现象描述。

## 第 10 步：验证救援通道（做完锁定测试再做）

这一步是验证"万一出问题能救回来"，很重要：

1. 手机上打开 Lockphone，点「**退出锁定**」
2. 电脑上执行：

```
C:\adb\platform-tools\adb.exe shell dpm remove-active-admin com.lockphone/.admin.LockAdminReceiver
```

预期输出：`Success: Admin removed`

3. **再把权限激活回来**（重复第 8 步的命令），确认再次输出 `Success`

**【记录】**两条命令的输出。

---

## 第 11 步：收尾

1. 最后再打开学习 APP 确认一次登录和数据正常
2. 把所有【记录】整理发回：每步的命令输出、第 9 步的表格、照片
3. 手机保持现状（已激活状态）即可，数据线可以拔掉

## 出问题的万能救援命令

手机被锁死退不出来、程序不正常时，连上电脑执行：

```
C:\adb\platform-tools\adb.exe shell dpm remove-active-admin com.lockphone/.admin.LockAdminReceiver
```

然后卸载程序：

```
C:\adb\platform-tools\adb.exe uninstall com.lockphone
```

手机即恢复原样（数据不受任何影响）。
