# Lockphone v3 复测指南（给现场操作人）

> 这一版在 v2 基础上加了**音量控制**（锁定音量 + 蓝牙音响时限 50-100%），其余修复沿用 v2。
> 规则不变：命令原样复制粘贴、结果记对应「输出结果」下、遇意外拍照停下。**绝不碰恢复出厂/清数据。**

**这一版新增/变化**：
- 家长设置里新增「**锁定音量**」开关
- 音量逻辑：**只管媒体音量**（放音乐/视频/音频课的音量），铃声、闹钟、通知音量不受影响
  - 开关**关**：音量随便调
  - 开关**开 + 连着蓝牙音响**：媒体音量只能在 **50%-100%** 之间（调到 50% 以下会自动弹回 50%）
  - 开关**开 + 没连蓝牙**：媒体音量被**冻结**（孩子按音量键会弹回原值，改不动）
- v2 的修复（改名/临时退出/PIN冷却/kiosk兜底）都在这一版里

## 第 1 步：装 v3

收到新 `app-debug.apk`，覆盖到 `C:\adb\app-debug.apk`，执行：

```
C:\adb\platform-tools\adb.exe install -t -r C:\adb\app-debug.apk
```

预期 `Success`，盯手机点安装确认。装完打开应直接进锁定桌面（数据、PIN、白名单保留）。
装好后手机可能出现一个「音量守护运行中」的常驻通知——这是正常的（音量控制需要它常驻），不用管。

## 第 2 步：音量控制（本版重点）

先进「家长模式」→ 设置页，确认有「**锁定音量（禁止调节音量）**」开关。

**A. 不连蓝牙、开关开（默认）：**
1. 保证没连蓝牙音响，开关是开的
2. 播放白名单里任意能出声的内容（多邻国的题目发音等）
3. 按音量加/减键调**媒体音量** → 预期：**改不动/弹回原值**
4. 试着调铃声（比如来电音量，一般在通话或单独入口）→ 预期：**能正常调**（铃声不受管）

### 输出结果 A
（媒体音量锁住了吗？铃声还能调吗？）

**B. 连蓝牙音响、开关开：**
1. 连上蓝牙音响
2. 放媒体，把媒体音量往下调 → 预期：**能调，但调到一半（50%）以下会自动弹回 50%**；往上能到 100%
3. 记录实际手感（弹回是否明显）

### 输出结果 B
（蓝牙下能调吗？低于 50% 弹回吗？）

**C. 开关关：**
1. 设置页把「锁定音量」关掉
2. 媒体音量 → 预期：**随便调，不再锁**
3. 测完把开关**开回来**

### 输出结果 C
（关掉后音量完全自由了吗？）

## 第 3 步：v2 的重点项复查（如果 v2 已测过且通过，这步可跳过，只补没测的）

> 关键提醒：**测 Home 键和重启时，全程不要点"临时退出"**（临时退出会清掉默认桌面身份）。

| # | 动作 | 预期 | 实际 |
|---|---|---|---|
| 1 | 应用名 | 显示「管住逆子的手」 | |
| 2 | 默认不旋转（锁竖屏开着） | 横过来不转 | |

| 4 | 重启手机 | 开机进锁定桌面 | |
| 5 | PIN 连错 5 次 + 反复转屏 | 冷却始终生效，转屏绕不过 | |
| 6 | 临时退出锁定（多试几次） | 每次都能稳定回 vivo 桌面 | |
| 7 | 重新打开 APP | 自动回锁定桌面 | |

## 第 4 步（可选）：限额检测诊断

> 这是为了解决「应用限额用完后点图标没反应、孩子不知道为什么」。先确认系统限额状态能不能被读到。

1. 未限额时，先跑一次记为「限额前」：
```
C:\adb\platform-tools\adb.exe shell "dumpsys package com.duolingo | grep -iE 'suspend|stopped|flag|enabled|hidden'"
```
2. 给多邻国设 1 分钟应用限时并用完，点图标确认没反应
3. 再跑一次记为「限额后」：
```
C:\adb\platform-tools\adb.exe shell "dumpsys package com.duolingo | grep -iE 'suspend|stopped|flag|enabled|hidden'"
```
4. 两次输出都发回

## 救援命令（出问题时用）
```
C:\adb\platform-tools\adb.exe shell dpm remove-active-admin com.lockphone/.admin.LockAdminReceiver
```
手机立即恢复自由，数据无损。
