# Lockphone v2 复测指南（给现场操作人）

> 上一轮验收辛苦了！根据你反馈的问题，程序做了一批修复。这一版**不用全部重测**，只测下面几个重点项。
> 规则不变：命令原样复制粘贴、结果记在对应「输出结果」下、遇到意外拍照停下。**绝不碰恢复出厂/清数据。**

**这一版改了什么**：
- 应用改名叫「**管住逆子的手**」
- 新增「锁定竖屏」开关（默认开，不再自动旋转）
- 修复了「临时退出锁定经常失效」
- 修复了「PIN 冷却转几次屏幕就被绕过」
- 新增被切走时自动抢回锁定桌面的兜底

---

## 第 1 步：装新版

收到新的 `app-debug.apk`，覆盖放到 `C:\adb\app-debug.apk`，手机连电脑执行：

```
C:\adb\platform-tools\adb.exe install -t -r C:\adb\app-debug.apk
```

预期 `Success`，执行后盯手机点安装确认弹窗。
（管理权限、之前设的 PIN 和白名单都会保留，装完打开应**直接进锁定桌面**，不用重设向导。如果反而要求重新设向导，先停下告诉我。）

## 第 2 步：改名 + 锁定竖屏开关

1. 看应用名/图标下方是不是变成了「**管住逆子的手**」
2. **不进设置**，直接横过来拿手机 → 屏幕应该**不旋转**（默认锁竖屏）
3. 进「家长模式」输 PIN → 设置页顶部应有「**锁定竖屏（关闭自动旋转）**」开关，默认是开的
4. 把它**关掉** → 返回桌面，横过来 → 这次应该能旋转了
5. 再进设置**开回来**（保持锁竖屏交给孩子用）

### 输出结果
（应用名？默认不旋转？开关能控制旋转？）

---

## 第 3 步：【重点】Home 键和重启（这次先别碰"临时退出"）

> 上次这两项失败，很可能是因为你测试中点过「临时退出锁定」，把默认桌面身份清掉了。这次**从装好到测完这一步，全程不要点临时退出**，直接测：

| # | 动作 | 预期 | 实际 |
|---|---|---|---|
| 1 | 在锁定桌面按 Home 键（你用的三键导航的房子键） | 留在锁定桌面，不去 vivo 桌面 | |
| 2 | 如果 #1 还是去了 vivo 桌面 | 观察：是不是**闪一下就被自动拽回**锁定桌面？（这是新兜底） | |
| 3 | 重启手机 → 开机解锁后 | 进入「管住逆子的手」锁定桌面（不是 vivo 桌面） | |

### 输出结果
（三项如实记录，尤其 #1 到底留住没、#2 有没有被拽回、#3 重启进哪）

---

## 第 4 步：PIN 冷却抗旋转（上次被绕过的）

1. 临时把「锁定竖屏」开关关掉（这样能转屏测试）
2. 点家长模式，**连输 5 次错误 PIN** → 提示要等 60 秒
3. 冷却期间**反复旋转手机 5~6 次**，每次转完再试着输错误 PIN
4. 预期：**始终**提示要等待，转多少次都绕不过
5. 测完把锁定竖屏开关**开回来**

### 输出结果
（转屏多次后冷却还在吗？）

---

## 第 5 步：临时退出修复（上次经常失效的）

1. 进设置页点「**临时退出锁定**」→ 预期这次能稳定回到 vivo 桌面（多试 2~3 次，看还会不会失效）
2. 重新打开「管住逆子的手」→ 预期**自动**回到锁定桌面

### 输出结果
（临时退出还失效吗？重开能自动锁定吗？）

## 救援命令（出问题时用）

```
C:\adb\platform-tools\adb.exe shell dpm remove-active-admin com.lockphone/.admin.LockAdminReceiver
```
手机立即恢复自由，数据无损。重新启用见预检指南第 8 步。
