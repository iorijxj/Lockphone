# Lockphone v4 复测指南（给现场操作人）

> 这一版针对 v3 反馈做了修复。重点测四项，其余沿用之前。
> 规则不变：命令原样复制粘贴、结果记对应「输出结果」下、遇意外拍照停下。**绝不碰恢复出厂/清数据。**

**这一版改了什么**（对应你上次的反馈）：
- **修复：白名单 APP 现在能正常用了**——上次"点任何应用都被拽回锁定桌面/多邻国进不去"，是一个兜底逻辑闯的祸，已移除
- **音量锁定值改成 70%**（无蓝牙锁死 70%，蓝牙下限 70%）
- **新增：开机自启**（尝试解决重启回默认桌面，可能要手动开自启动权限，见第 3 步）
- **新增：限额提示**——白名单 APP 打不开时弹「该应用今日可能已达使用限额」

---

## 第 3 步：重启 + 开机自启

1. 先去开自启动权限：手机 设置 → 应用管理 → 找到「管住逆子的手」→ 权限/自启动 里，**允许自启动/后台运行**（vivo 一般在 i管家 或 设置→应用→自启动管理）
2. 重启手机 → 开机解锁后看进哪里

| 动作 | 预期 | 实际 |
|---|---|---|
| 开了自启动后重启 | 开机进入锁定桌面 | |

### 输出结果
（找到自启动开关了吗？开了之后重启进锁定桌面还是 vivo 桌面？）

---

## 第 4 步：音量 70%

1. 不连蓝牙、开关开：调媒体音量 → 预期锁死在 **70%**
2. 连蓝牙音响、开关开：媒体音量能在 **70%-100%** 调，低于 70% 弹回
3. 开关关：音量自由

### 输出结果
（无蓝牙锁在 70% 吗？蓝牙下限是 70% 吗？）

---

## 第 5 步：限额提示（新）

1. 给多邻国设 1 分钟应用限时，用完（让它进入"打不开"状态）
2. 在锁定桌面点多邻国图标
3. 预期：等约 1~2 秒后，弹出「该应用今日可能已达使用限额」提示
4. 另外：点一个**正常没限额**的白名单 APP，确认它**能正常打开、不会误弹**这个提示

### 输出结果
（限额的 APP 点了会弹提示吗？正常 APP 会不会误弹？）

## 救援命令
```
C:\adb\platform-tools\adb.exe shell dpm remove-active-admin com.lockphone/.admin.LockAdminReceiver
```
手机立即恢复自由，数据无损。
