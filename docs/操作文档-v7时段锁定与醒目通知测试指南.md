# v7 测试指南：自定义可用时间段 + 受限 APP 醒目限额通知

本轮新增两个功能：

1. **自定义可用时间段（Curfew）**：非可用时段全屏锁定「非授权使用时间段」，家长 PIN 验证后可现场选择临时解锁时长。
2. **受限 APP 醒目限额通知**：受限 APP 每次切到前台弹 heads-up 通知（限额/剩余分钟），剩余 ≤5 分钟逐档预警也改为醒目通知（替代原 Toast）。

## 0. 安装

```
adb install -r -t app-debug.apk
```

升级安装即可，无需重新激活 Device Owner；旧配置（白名单、PIN、限时）全部保留。

## 1. 通知权限自动授予（Android 13+）

1. 安装后进入锁定桌面一次（触发 applyPolicies）。
2. 验证：`adb shell dumpsys package com.lockphone | findstr POST_NOTIFICATIONS`，应看到 `granted=true`。
3. 全程**不应**弹出系统的通知权限请求框。

## 2. 受限 APP 激活通知

前置：家长设置里给某 APP（如抖音）开限时，设 3 分钟。

| 步骤 | 预期 |
|------|------|
| 从锁定桌面点开该 APP | 顶部弹出 heads-up 通知「『XX』限时 3 分钟 / 今天还剩 3 分钟可用」，带振动 |
| 停留继续使用 | 剩余进入 2 分钟、1 分钟档位时各弹一次「只剩 N 分钟了」预警通知 |
| 时间用完 | APP 被挂起并踢回桌面（原有行为），图标置灰 |
| 退回桌面再点开另一个受限 APP | 弹该 APP 自己的激活通知 |
| 反复切进切出同一 APP | 每次进入弹一次；下拉通知栏同一 APP 始终只有一条（覆盖更新，不堆叠） |

注意：激活通知最多有 ~5 秒延迟（守护服务 5 秒轮询一次）。

## 3. 可用时间段锁定

前置：家长设置 → 打开「限定可用时间段」开关 → 添加时段。时间格式为 HH:mm（如 08:00），格式不合法不会保存。

### 3.1 基本锁定

1. 设一个**不含当前时刻**的时段（如现在 15:00，设 20:00-21:00）。
2. 返回桌面 → ≤5 秒内桌面变为深色全屏「非授权使用时间段」锁定页，显示「下次可用：20:00」，无任何 APP 图标。
3. 反过来设一个包含当前时刻的时段 → 桌面正常显示。

### 3.2 使用中被踢回

1. 孩子正在白名单 APP 里使用。
2. 到达时段结束点（可设一个 1-2 分钟后结束的时段现场等）。
3. 预期：≤5 秒内被强制踢回锁定页。

### 3.3 家长临时解锁

1. 锁定页点「家长模式」→ 输入 PIN。
2. 弹出时长选择：15 / 30 / 60 分钟 / 直到下个可用时段 / 打开完整设置。
3. 选 15 分钟 → 立即恢复正常桌面，APP 可用；15 分钟后 ≤5 秒内自动重新锁定。
4. 再次解锁选「直到下个可用时段」→ 到下个时段开始点自然衔接，全程无需再干预。
5. 「打开完整设置」→ 直达家长设置页，可改时段/关总开关。

### 3.4 组合与边界

| 场景 | 预期 |
|------|------|
| 多条时段（如 08:00-12:00 + 14:00-20:00） | 两段内都可用，12:00-14:00 之间锁定 |
| 跨午夜时段（如 20:00-06:00） | 23:59 → 00:00 跨天不中断可用 |
| 关闭总开关 | 立即（≤5 秒）恢复全天可用 |
| 删除所有时段（开关仍开） | 全天可用（空列表=不限制） |
| 临时解锁期间使用受限 APP | 配额照常计时、照常挂起（解锁只放行时段，不豁免限时） |

### 3.5 重启存活

1. 处于非可用时段时重启手机 → 开机进入桌面后 ≤5 秒变锁定页。
2. 临时解锁 60 分钟后立刻重启 → 开机后仍处于解锁状态（截止时间跨重启生效），到点重新锁定。

## 4. 回归检查

- 原有功能不受影响：白名单增删、限时/加时、锁竖屏、锁音量、临时退出锁定、彻底解除、密码错误记录。
- 配额挂起/跨天清零照旧（v6 验收项抽查 2-3 条即可）。

## 已知边界（可接受）

- 时段锁定/解锁生效有 ≤5 秒轮询延迟（与限时挂起同粒度）。
- 被踢回前的几秒仍计入该 APP 配额（既有轮询粒度）。
- 守护服务被系统杀死重建后，若孩子仍停留在受限 APP 内，可能重复弹一次激活通知（无害）。
- 若在系统设置里手动关闭本 APP 的通知总开关，限额通知会静默失效（不影响锁定/挂起等核心管控）。
