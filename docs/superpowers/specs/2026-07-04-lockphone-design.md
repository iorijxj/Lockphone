# Lockphone 儿童锁定桌面 — 设计文档

日期：2026-07-04
状态：已与用户确认

## 1. 背景与目标

给自家孩子的一台闲置 OPPO/vivo 系安卓手机做一个锁定桌面 APP：

- 手机只显示家长允许的白名单 APP，白名单内孩子可自由使用
- 未经家长授权（PIN）无法退出锁定环境
- 仅自用，不发布、不上架
- **约束：手机上已有在用的学习 APP，激活过程不能清除数据**（不恢复出厂设置）

时间管理不在本 APP 范围内——由 ColorOS/OriginOS 系统自带的「应用限时」功能承担（已确认此决定，砍掉了原计划的计时监控模块）。

## 2. 技术路线

**自研自定义 Launcher + Device Owner + Lock Task Mode**（Android 官方 kiosk 方案）。

不恢复出厂即可激活 Device Owner：真正的前提是激活时设备上没有已登录的系统级账号、没有多用户。流程为「删除系统账号 → 开 USB 调试 → `adb shell dpm set-device-owner` 激活」，所有 APP 及其数据原封不动。国内学习类 APP 基本使用应用内自有账号，删除手机系统账号不影响其登录态（Phase 0 中实测确认）。

已否决的备选：
- 系统自带儿童空间——用户已实测，不满足需求
- 开源项目改造——无完整匹配项目，改造不比自研快（TestDPC 仅作 API 参考）
- 商业 MDM——付费且不贴合儿童自用场景

## 3. 总体架构

单个 APK，三个模块（原计划的计时监控模块已砍掉）：

```
┌─────────────────────────────────────────┐
│  Lockphone APK                          │
│                                         │
│  1. 锁定桌面 (Launcher)                  │
│     注册为系统 HOME，网格展示白名单 APP   │
│     顶部/底部有明显的「家长模式」按钮      │
│                                         │
│  2. 设备管理核心 (Device Owner)          │
│     Lock Task Mode 白名单锁定            │
│     封堵各系统级逃逸口                    │
│                                         │
│  3. 家长设置页 (PIN 保护)                │
│     白名单管理 / 改 PIN /                │
│     临时退出锁定 / 彻底解除               │
└─────────────────────────────────────────┘
```

数据全部本地存储（DataStore），无网络、无服务器。

## 4. 锁定机制

### 4.1 一次性激活（家长在电脑上操作）

1. `adb shell dumpsys account` 列出系统账号，在设置中逐个删除（OPPO/vivo 账号可能需先关「查找手机」）
2. 开启 USB 调试
3. `adb shell dpm set-device-owner <package>/<AdminReceiver>` 激活

### 4.2 锁定生效

- Device Owner 将本 APP 设为固定 HOME（`addPersistentPreferredActivity`），按 Home 键永远回到锁定桌面，无桌面选择弹窗
- `setLockTaskPackages(白名单 + 自身)`，桌面启动后进入 Lock Task Mode

### 4.3 逃逸口封堵

| 逃逸口 | 封堵方式 |
|---|---|
| 返回键 / Home 键 | 回到锁定桌面 |
| 最近任务键 | Lock Task 下系统禁用 |
| 下拉通知栏 / 快捷设置 | Lock Task 下系统禁用 |
| 启动白名单外 APP | 系统拒绝启动 |
| 重启手机 | 固定 HOME，开机自动进入锁定桌面 |
| 安全模式 | `DISALLOW_SAFE_BOOT` 用户限制 |
| 恢复出厂设置 | `DISALLOW_FACTORY_RESET` 用户限制（设置 APP 本身也不在白名单） |
| 卸载本 APP | Device Owner 不可卸载 |

### 4.4 USB 调试策略（已确认）

**激活后保留 USB 调试。** 理由：APP 出 bug 或家长忘记 PIN 时可通过电脑 adb 救援，不会把自己锁死。代价（孩子若懂电脑且能拿到数据线理论上可绕过）已知悉并接受。

## 5. 家长设置页

### 5.1 入口（已确认：明面按钮，非隐藏手势）

锁定桌面上有明显的「家长模式」按钮 → 弹出 PIN 输入框（二级密码确认）→ 验证通过进入设置页。

### 5.2 PIN

- 首次启动向导中设置；本地仅存加盐哈希，不存明文
- 连错 5 次冷却 60 秒，防穷举
- 忘记 PIN 兜底：电脑 adb 解除（依赖 4.4 保留调试的决定）

### 5.3 功能（四项）

1. **白名单管理**：列出已装 APP（带图标），勾选上桌面/取消隐藏，改动即时生效（同步更新 `setLockTaskPackages` 与桌面）
2. **修改 PIN**
3. **临时退出锁定**：家长自用手机时退出到系统原桌面；恢复方式为重新打开本 APP（自动重新进入锁定桌面）
4. **彻底解除**：一键放弃 Device Owner，手机完全恢复原状（确认弹窗防误触），数据不受影响

## 6. 数据与首次启动

- 存储内容仅两项：PIN 哈希 + 白名单包名列表，用 DataStore，无数据库
- 首次启动向导（激活 Device Owner 后第一次打开）：设 PIN → 勾选白名单 → 确认进入锁定，共 3 步

## 7. Phase 0 真机预检（写正式代码之前，约半天）

用最小验证 APK 在目标手机上按顺序确认：

1. `dumpsys account` 列账号 → 删除 → **确认学习 APP 登录态未受影响**
2. `dpm set-device-owner` 能否激活成功 ← **全项目最大不确定点**
3. Lock Task Mode 生效：通知栏、最近任务、白名单外 APP 启动均被封
4. 重启后自动回到锁定桌面
5. 系统自带「应用限时」在锁定桌面下依然生效
6. `DISALLOW_SAFE_BOOT` 等用户限制在此 ROM 上有效

**降级路径**：若第 2 条失败（ROM 拦截激活），停止本方案，另行设计软锁路线（普通 Launcher + 屏幕固定 + 无障碍服务），锁定强度打折但数据同样安全。软锁方案届时单独出设计，不在本文档范围。

## 8. 异常处理

- **桌面 APP 崩溃**：作为固定 HOME 会被系统自动重新拉起，天然自愈，无需看门狗
- **白名单 APP 拉起名单外页面**（如 APP 内跳浏览器）：系统静默拦截，表现为点击无反应；确有需要时家长将目标 APP 加白
- **APP 更新**：`adb install -r` 覆盖安装，配置与锁定状态保留

## 9. 测试策略

按自用小工具标准裁剪（参照个人 testing 规则）：

- 单元测试：仅覆盖纯逻辑（PIN 加盐哈希与校验、冷却计数、白名单存取）
- 主体验证：真机手动验收——第 7 节 6 条预检项 + 「孩子视角攻击清单」（长按电源键、插拔数据线、疯狂点返回、通知栏边缘滑动、白名单 APP 内跳转逃逸等逐项实测）

## 10. 技术栈与工作量预估

- Kotlin + Jetpack Compose（桌面网格 + 设置页 UI 简单，Compose 开发效率最高）
- minSdk 按目标手机实际系统定（Phase 0 时确认），仅适配这一台设备
- 预估：Phase 0 半天；正式开发（桌面 1-2 天、Device Owner 集成 1 天、设置页 1 天、真机联调 1-2 天）合计约 1 周业余时间，用户基本靠 AI 代写
