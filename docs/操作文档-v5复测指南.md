# Lockphone v5 复测指南（给现场操作人）

> v4 的问题基本都解决了。这一版只加了两个小功能，测一下即可。
> 规则不变：命令原样复制粘贴、结果记对应「输出结果」下、遇意外拍照停下。**绝不碰恢复出厂/清数据。**

**这一版新增**：
- **密码错误记录**：每次家长验证输错 PIN 都会记下时间，设置页可查看
- **PIN 最长支持 256 位**（之前上限 8 位）

**已知不改**：Home 键在 vivo 锁定模式下无效（系统限制），**回锁定桌面请用返回键**——这是确认接受的方式，不是 bug。

---

## 第 1 步：装 v5

```
C:\adb\platform-tools\adb.exe install -t -r C:\adb\app-debug.apk
```
预期 `Success`，盯手机点确认。装完打开进锁定桌面（数据/PIN/白名单保留）。

### 输出结果
通过

---

## 第 2 步：密码错误记录

1. 进「家长模式」，**故意输错 PIN 2~3 次**（记下大概时间）
2. 用正确 PIN 进设置页
3. 点「**密码错误记录**」按钮
4. 预期：弹出列表，标题显示「密码错误记录（共 N 次）」，下面按**最新在上**列出刚才输错的**日期时间**（精确到秒，如 2026-07-05 09:30:15），时间对得上

### 输出结果
（错误次数和时间对得上吗？）
看不到「**密码错误记录**」按钮 
---

## 第 3 步：PIN 最长 256 位

1. 设置页「修改 PIN」，设一个**比较长的 PIN**（比如输十几二十位数字，验证不再卡在 8 位）
2. 两次输入一致，保存
3. 退出后用这个长 PIN 进家长模式，能进
4. （测完可改回你平时用的 PIN）

### 输出结果
（能设超过 8 位的长 PIN 吗？长 PIN 能正常验证吗？）

通过
---

## 第 4 步：回桌面用返回键（确认体验可接受）

1. 打开一个白名单 APP，用一会儿
2. 用**返回键**回到锁定桌面（多点几下直到回桌面）
3. 确认能回、孩子能接受这个操作

### 输出结果
（返回键回桌面顺畅吗？）
顺畅

---

## 第 5 步：收尾
1. 多邻国确认登录和进度正常
2. 结果发回
3. 手机保持锁定交给孩子正常使用

### 输出结果
（在这里写）
正常
---

## 救援命令
```
C:\adb\platform-tools\adb.exe shell dpm remove-active-admin com.lockphone/.admin.LockAdminReceiver
```
手机立即恢复自由，数据无损。
